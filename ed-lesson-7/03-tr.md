# 🔁 Topic 3 — Character Replacement with `tr`

> **`tr` translates or deletes individual characters — the simplest way to swap one character for another.**

← [Prev: sed](./02-sed.md) · [Next: cut →](./04-cut.md)

---

## What is `tr`?

`tr` stands for **translate**.

It reads text and replaces or removes specific characters.
Unlike `sed`, it works on single characters — not words or patterns.

---

## Important — `tr` reads from stdin, not a file

`tr` does not take a filename as an argument. You must use `<` or a pipe:

```bash
# Using < to feed a file into tr
tr 'a' 'b' < file.txt

# Using a pipe
cat file.txt | tr 'a' 'b'
```

---

## Basic syntax

```bash
tr 'FIND' 'REPLACE' < filename
```

| Part | Meaning |
|------|---------|
| `FIND` | the character(s) to look for |
| `REPLACE` | the character(s) to put in instead |

---

## Simple examples

```bash
# Replace every comma with a space
tr ',' ' ' < table.csv

# Replace every space with a comma
tr ' ' ',' < femalenames.txt

# Replace all lowercase letters with uppercase
tr 'a-z' 'A-Z' < file.txt

# Replace all uppercase letters with lowercase
tr 'A-Z' 'a-z' < file.txt
```

---

## The `-s` flag — squeeze repeated characters

`-s` squeezes multiple repeated characters into one.

This is very useful for cleaning files with irregular spacing:

```bash
tr -s ' ' < file.txt         # squeeze many spaces into one space
tr -s ' ' ',' < file.txt     # squeeze many spaces, then replace with comma
```

Before:
```
MARY       2.629   2.629    1
```

After `tr -s ' ' ','`:
```
MARY,2.629,2.629,1
```

---

## The `-d` flag — delete characters

```bash
tr -d '\n' < file.txt        # delete all newlines (joins all lines into one)
tr -d ' ' < file.txt         # delete all spaces
tr -d ',' < file.csv         # delete all commas
tr -d '0-9' < file.txt       # delete all digits
```

---

## Save the result to a file

```bash
tr -s ' ' ',' < femalenames.txt > table.csv
```

---

## tr vs sed — which to use?

| Situation | Use |
|-----------|-----|
| Replace one character with another | `tr` — simpler |
| Replace a pattern (e.g. one or more spaces) | `sed` — more powerful |
| Delete a specific character | `tr -d` |
| Remove leading/trailing whitespace | `sed` |

For the skills test, `sed 's/\s\+/,/g'` is more reliable than `tr` for cleaning files — because `tr -s ' '` only handles spaces, not tabs.

---

## Full example — two ways to clean the same file

```bash
# Method 1 — sed (handles tabs AND spaces)
sed 's/\s\+/,/g' femalenames.txt > table.csv

# Method 2 — tr (handles spaces only)
tr -s ' ' ',' < femalenames.txt > table.csv
```

Both produce the same result when the file uses spaces.
